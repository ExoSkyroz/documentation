How does HTML, CSS and JavaScript work?


total.js is doing the dirty job for you. Its core has implemented good tools to improve development
and rendering web site for all web browsers.

In release mode ...

  • Framework sets reasonable HTTP cache for static files (IMG, JS, CSS).
  • Framework clears used resources in a time loop.
  • Release mode returns highly rated content.
@{image('pagespeed.png', 751, 73, 'PageSpeed')}

HTML

  • All comments are removed from HTML.
  • All HTML code is minified under compilation.
@{image('source-code.gif', 810, 300, 'total.js rendered HTML code')}

CSS

  • All CSS code is minified (by default true) framework.config['allow-compile-css']
  • CSS supports - auto vendor prefixes - auto vendor prefixes example.
  • CSS supports - LESS (supports only constants and functions) - LESS example.
  • You can disable auto-vendor-prefixes - remove the mark from CSS: /*auto*/
  • All inline code in HTML <style type="text/css">... is compiled and minified.
  • You can rewrite built-in compilation - SASS example.
  • If the filename contains ".min" then it will be not compiled.
@{image('css.png', 725, 766, 'CSS')}

JavaScript

  • All JavaScript code is minified (by default true) framework.config['allow-compile-js']
  • All inline code in HTML <script type="text/javascript">... is minified.
  • You can rewrite built-in minification - UglifyJS example.
  • If the filename contains ".min" then it will be not compiled.
